Git에서 **Stash(스태시)**는 하던 작업을 잠시 옆으로 치워두고 싶을 때 사용하는 아주 유용한 기능입니다. 

비유하자면, 서류 작업을 하다가 급한 다른 일이 생겼을 때, 책상 위에 펼쳐진 서류들을 **'임시 보관함'**에 쓱 밀어 넣어 책상을 깨끗하게 비우는 것과 같습니다. 나중에 다시 보관함에서 꺼내면 원래 하던 작업 위치 그대로 되돌아옵니다.

---

### ## 1. 언제 사용하나요?
* **급한 수정 요청:** 기능 구현 중에 갑자기 긴급한 버그를 수정해야 해서 브랜치를 바꿔야 할 때.
* **코드 실험:** 현재 코드를 날리기는 아깝지만, 다른 방식으로 코드를 짜보고 싶을 때.
* **Pull 받기 전:** 원격 저장소의 내용을 가져오고 싶은데 내 로컬 수정 사항과 충돌이 날 것 같을 때.

---

### ## 2. 주요 명령어
가장 자주 쓰이는 명령어들입니다.

| 명령 | 설명 |
| :--- | :--- |
| `git stash` | 현재 작업(Modified & Staged)을 저장하고 워킹 디렉토리를 깨끗하게 만듭니다. |
| `git stash save "메시지"` | 스태시를 만들 때 나중에 알아보기 쉽게 이름을 붙입니다. |
| `git stash list` | 저장된 스태시 목록을 확인합니다. |
| `git stash apply` | 가장 최근의 스태시를 다시 불러옵니다. (목록에는 유지됨) |
| `git stash pop` | 가장 최근의 스태시를 불러오면서, 목록에서 삭제합니다. |
| `git stash drop` | 필요 없는 특정 스태시를 삭제합니다. |

---

### ## 3. 주의사항
1.  **추적되지 않는 파일(Untracked files):** 새롭게 만든 파일(git add 안 된 파일)은 기본적으로 스태시에 포함되지 않습니다. 포함하려면 `-u` 옵션을 써야 합니다.
    > `git stash -u`
2.  **Conflict(충돌):** 스태시를 다시 불러올 때, 그 사이 바뀐 코드와 겹치면 충돌이 날 수 있습니다. 이럴 땐 일반적인 머지(Merge)처럼 해결해주면 됩니다.
