네, 당연히 바꿀 수 있습니다! 수학 문제를 풀다가 마지막 답안지에 이름을 안 썼거나, 기호를 잘못 적어서 다시 수정하는 것과 같아요.

상황에 따라 **두 가지 방법**이 있습니다. 이미 GitHub에 **`push`를 했는지 안 했는지**가 중요합니다.

---

### 1. 아직 `push` 하기 전이라면? (가장 쉬움)

내 컴퓨터에서만 커밋을 한 상태라면, 방금 한 커밋 메시지를 바로 덮어쓸 수 있습니다.

* **터미널 명령어:**
  ```bash
  git commit --amend -m "새로운 커밋 메시지 내용"
  ```
* **인텔리제이 활용:**
  1. `Commit` 탭(Alt + 0)을 엽니다.
  2. 아래쪽 **`Amend`** 체크박스를 클릭합니다.
  3. 메시지를 다시 쓰고 커밋 버튼을 누르면 이전 메시지가 사라지고 새 메시지로 바뀝니다.

---

### 2. 이미 GitHub에 `push`까지 해버렸다면? (주의!)

이때는 좀 조심해야 합니다. 이미 보관함(GitHub)에 들어간 기록을 강제로 수정해서 다시 던져야 하거든요.

1. 우선 위와 똑같이 **`git commit --amend -m "수정할 메시지"`**를 해서 내 컴퓨터 기록을 바꿉니다.
2. 그다음, GitHub에 강제로 덮어씌워야 합니다.
   ```bash
   git push origin main --force
   ```
   *(주의: 혼자 하는 프로젝트라면 상관없지만, 팀 프로젝트라면 다른 팀원들의 기록과 꼬일 수 있어 위험한 명령어입니다. 지금은 혼자 하시니까 괜찮아요!)*

---

### 3. "누락"이라는 게 메시지 내용인가요, 아니면 파일을 빠뜨린 건가요?

만약 메시지가 아니라 **"아 맞다! 아까 그 파일도 같이 커밋했어야 하는데!"** 하고 파일을 빼먹은 거라면 이렇게 하세요.

1. 빠뜨린 파일을 `git add` 합니다.
2. **`git commit --amend --no-edit`** 명령어를 칩니다.
3. 그러면 메시지는 그대로 두고, 방금 추가한 파일만 이전 커밋에 슬쩍 끼워 넣어줍니다. (아주 유용한 꿀팁입니다!)
