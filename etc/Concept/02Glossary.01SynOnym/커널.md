- WSL2 같은 방식은 윈도우 OS '아래'에 있는 '하이퍼바이저(Hypervisor)'라는 판 위에서 윈도우와 리눅스가 나란히 달리는 구조라고 보시면 됩니다.
- 커널.하이퍼바이저
- 커널.`하이퍼바이저`
- 커널.`하이퍼바이저`.이부분에서 OSㆍ리눅스 택일형태
- 예전에는 윈도우가 '주인'이고 리눅스가 그 위에서 돌아가는 '손님' 같았다면, 지금은 구조가 완전히 바뀌었습니다.
- 하이퍼바이저 (Hyper-V): 컴퓨터 하드웨어 바로 위에 깔리는 가장 얇고 강력한 관리 층입니다.
- 윈도우 OS & 리눅스 커널: 이 하이퍼바이저 위에서 동등한 층위로 돌아갑니다.
- 윈도우 vs 리눅스 커널의 관계 (WSL2 기준)
- 즉, 리눅스 명령어를 입력하면 윈도우 커널을 거쳐서 내려가는 게 아니라, 리눅스 커널이 하드웨어 자원을 직접 (하이퍼바이저를 통해) 효율적으로 사용하게 됩니다. 
- 윈도우 OS의 간섭을 덜 받으니 당연히 "아싸리" 속도가 빨라지는 거죠.
- Git Bash 같은 툴은 말씀하신 대로 내부에서 윈도우가 알아먹게끔 "이건 윈도우의 이 기능이야"라고 매핑(Mapping) 해주는 과정을 거칩니다. 
- 반면 WSL은 번역 없이 그냥 리눅스 그 자체로 실행됩니다.

- Git Bash: 윈도우라는 거대한 성벽 안에서 리눅스 말투를 흉내 내는 것 (번역 필요).
- WSL2: 윈도우 성벽 옆에 리눅스 전용 성을 따로 짓고, 하드웨어라는 땅을 같이 나눠 쓰는 것.
- 도커(Docker): 리눅스 커널 기능이 필수인데, WSL2 덕분에 윈도우에서도 네이티브 속도로 돌아갑니다.
