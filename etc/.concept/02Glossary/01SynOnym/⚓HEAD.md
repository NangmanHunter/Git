- HEAD는 현재 내가 위치해 있는 최신 커밋을 가리키는 포인터(화살표)입니다.
- "지금 내 작업 트리의 주인공은 이 커밋이야!"라고 알려주는 표식이죠.
- 보통 가장 최근에 한 커밋을 가리키고 있습니다.
- Git은 기본적으로 '대문자'가 표준입니다
- HEAD
- `HEAD`
- `HEAD`.`Head`
- `HEAD`.`Head`.`head`
- 표준: 원래는 대문자 HEAD가 맞습니다.
- 이유: 지금 사용 중인 OS(macOS/Windows)의 파일 시스템이 대소문자를 구별하지 않고 찾아주기 때문입니다.
- 결론: 리눅스 서버와의 호환성을 위해 가급적 **대문자 HEAD**로 쓰는 습관을 들이는 것이 좋습니다!
- 왜 소문자 head도 알아먹을까? A. 운영체제 파일 시스템의 관대함 (macOS, Windows)👉질문자님이 현재 macOS나 Windows를 사용 중이시라면, 이 시스템들은 기본적으로 대소문자를 구분하지 않는(Case-insensitive) 파일 시스템을 사용합니다.
- 왜 소문자 head도 알아먹을까? B. Git의 자체적인 유연성👉설령 대소문자를 구분하는 순수 리눅스 환경이라 하더라도, 최신 Git 버전들은 사용자의 편의를 위해 입력된 명령어를 내부적으로 해석할 때 head를 HEAD로 자동 매핑해주는 처리를 하기도 합니다. 하지만 이는 환경 설정이나 Git 버전에 따라 다를 수 있습니다.
